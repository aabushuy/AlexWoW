# M6.9 — Управление инвентарём (перемещение/сплит/своп/экипировка)

**Статус:** ⬜ запланировано
**Зависит от:** M6.1 (предметы/инвентарь), M6.2 (вендор — частично пересекается).
**Эталон:** CMaNGOS-WotLK (`Player::SwapItem`, `Player::SplitItem`, `Player::EquipItem`,
`PlayerStorage`).

## Цель / критерий приёмки (клиент)

Сейчас предметы только отображаются (M6.1) и появляются/уходят при торговле (M6.2), но
**манипулировать** ими нельзя: перетаскивание между слотами, **сплит стака**, своп, экипировка
кликом, выброс — не работают (сервер не обрабатывает соответствующие CMSG).

Нужно: перетащить предмет в другой слот; **разделить стопку** (split) на N в пустой слот;
поменять два предмета местами; экипировать/снять предмет (right-click / drag в слот экипировки);
выбросить предмет. Всё отражается у клиента и **персистентно**.

## Протокол (3.3.5a — опкоды из локальной wow_messages)
- `CMSG_AUTOEQUIP_ITEM 0x10A` — экипировать предмет (right-click / в слот экипировки).
- `CMSG_AUTOSTORE_BAG_ITEM 0x10B` — переместить в сумку.
- `CMSG_SWAP_ITEM 0x10C` — поменять местами (между контейнерами).
- `CMSG_SWAP_INV_ITEM 0x10D` — поменять/переместить в основном контейнере.
- `CMSG_SPLIT_ITEM 0x10E` — разделить стопку (src bag/slot → dst bag/slot, count).
- `CMSG_AUTOEQUIP_ITEM_SLOT 0x10F` — экипировать в конкретный слот.
- `CMSG_DESTROYITEM 0x111` — выбросить/уничтожить предмет.
- Ответы об ошибках — `SMSG_INVENTORY_CHANGE_FAILURE` (коды: нет места, нельзя экипировать и т.п.).

## Задачи
- `InventoryHandlers.cs`: разбор перечисленных CMSG; пересчёт `character_items` (slot/bag/stack);
  отправка обновлений (item-объекты / `PLAYER_FIELD_INV_SLOT` / `ITEM_FIELD_STACK_COUNT`).
- Сплит: уменьшить stack у источника (item VALUES-апдейт `ITEM_FIELD_STACK_COUNT`), создать новый
  item в целевом слоте на `count`.
- Своп/перемещение: обновить слоты двух предметов + `PLAYER_FIELD_INV_SLOT`.
- Экипировка: валидация по `InventoryType` (см. `InventorySlots.EquipSlotFor`), перенос в слот
  экипировки + видимые предметы (`PLAYER_VISIBLE_ITEM_*`).
- Выброс: удалить item (как продажа без денег) + DESTROY.
- Валидация и `SMSG_INVENTORY_CHANGE_FAILURE`.
- Серверные ITEM VALUES-апдейты (`ITEM_FIELD_STACK_COUNT`) — добавить хелпер в `ItemObject`.

## Проверка
Живой клиент: перетащить предмет в пустой слот; разделить стопку (5→2+3); поменять два местами;
экипировать оружие кликом и снять; выбросить предмет; релог — раскладка сохранена.
