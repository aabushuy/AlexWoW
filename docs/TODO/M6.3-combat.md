# M6.3 — Бой (мили-атака по NPC)

**Статус:** ⬜ запланировано
**Зависит от:** M5 (видимость NPC), вводит **серверный тик** и **авторитетные сущности**.
**Эталон:** CMaNGOS-WotLK (`Unit::AttackerStateUpdate`, `Unit::CalculateMeleeDamage`,
`Unit::DealDamage`, swing-таймеры).

## Цель / критерий приёмки (клиент)

Выбрать враждебного NPC, авто-атака → плывут числа урона, HP NPC падает, NPC **умирает**
(труп), затем **респавнится**. (Ответный удар NPC и преследование — M6.7.)

## Архитектурный фундамент (вводится здесь)

Сейчас NPC — per-session immutable `NpcSpawn`, серверного цикла нет. Для боя нужно:
- **Авторитетные сущности существ** (`WorldCreature`: мутабельный Health/MaxHealth, alive/dead,
  RespawnAt) в реестре `WorldState` (общие для всех наблюдателей; лениво создаются из тех же
  DB-строк, что и видимость). `VisibleNpcs` хранит ссылку на общий `WorldCreature`.
- **Серверный тик** (`WorldUpdateLoop : BackgroundService`, ~250 мс): swing-таймеры, смерть, респавн,
  (позже — реген, ИИ). Рассылка изменений HP/смерти наблюдателям (игроки в радиусе на карте).
- **Реальные HP**: игрок (формула по классу/уровню — упрощённо, точные статы позже), NPC
  (из `creature_template` / `creature_classlevelstats` × HealthMultiplier — сверить схему `mangos`).

## Протокол (3.3.5a — подтверждено gtker/эталонами)
- `CMSG_SET_SELECTION = 0x13D` (packed guid цели) — хранить target.
- `CMSG_ATTACKSWING = 0x141` (u64 victim, **не** packed) → `SMSG_ATTACKSTART = 0x143`
  (u64 attacker + u64 victim). Старт авто-атаки, swing-таймер ~2000 мс.
- `CMSG_ATTACKSTOP = 0x142` → `SMSG_ATTACKSTOP = 0x144` (packed player + packed enemy + u32 0).
- `SMSG_ATTACKERSTATEUPDATE = 0x14A`: hit_info(u32) + attacker(packed) + target(packed) +
  total_damage(u32) + overkill(u32) + count(u8=1) + [school(u32)+dmgF(f32)+dmg(u32)+absorb(u32)+
  resist(u32)] + victim_state(u8=1) + unk1(u32) + unk2(u32). Без absorb/resist/block-флагов —
  conditional-поля опускаются. HitInfo для обычного удара — сверить клиентом (≈ NORMALSWING2=0x2).
- Изменение HP — partial `SMSG_UPDATE_OBJECT` (UPDATETYPE_VALUES=0): packed guid + маска с
  `UNIT_FIELD_HEALTH`. Смерть — health=0 (клиент проигрывает death). Лут — M6.6.

## Задачи
- `WorldCreature` + реестр в `WorldState`; перевод видимости на авторитетные сущности.
- `WorldUpdateLoop` (hosted service) + `WorldState.Update(dt)`.
- `CombatHandlers.cs`: set-selection / attackswing / attackstop.
- Расчёт урона (упрощённо), смерть + респавн (таймер), рассылка наблюдателям.
- HP игрока/NPC в спавне (`PlayerSpawn`/`CreatureUpdate`) из реальных значений.
- `BuildValuesUpdate(guid, mask)` — хелпер partial-update.

## Проверка
Живой клиент: таргет на враждебного NPC (волк у старта) → авто-атака → числа урона, HP падает,
NPC умирает (труп) → через ~30 с респавн. Второй клиент видит бой/смерть.
