# M6.7 — ИИ существ (aggro, преследование, ответный бой)

**Статус:** ⬜ запланировано
**Зависит от:** M6.3 (бой, авторитетные сущности, тик), mmaps (навмеш — готов в M5.5+).
**Эталон:** CMaNGOS-WotLK (`CreatureAI`, `MotionMaster`, `ThreatManager`, `MovementGenerator`).

## Цель / критерий приёмки (клиент)

Враждебное существо **агрится** на подошедшего игрока, поворачивается, **преследует** по
навмешу (mmaps — обход препятствий), бьёт в ближнем бою; игрок получает урон и **может умереть**.
Вне зоны/при отрыве — существо возвращается на спавн (evade) и регенит HP.

## Фундамент
- **Потребитель mmaps**: `Navmesh.FindPath` (готов) → путь существа к цели; ститчинг тайлов
  (межтайловые пути) — доработать при необходимости.
- Движение существа — рассылка `MSG_MOVE_*`/`SMSG_MONSTER_MOVE` наблюдателям (сплайн-движение).

## Протокол (3.3.5a — сверить с gtker)
- `SMSG_MONSTER_MOVE` (сплайн к точкам пути), `SMSG_AI_REACTION` (aggro), фейс/ориентация.
- Ответная атака — те же `SMSG_ATTACKSTART`/`SMSG_ATTACKERSTATEUPDATE` (из M6.3) в сторону игрока.
- Смерть игрока: `SMSG_UPDATE_OBJECT` (health=0), `CMSG_REPCLAIM`/возрождение — отдельно.

## Задачи
- В тике (M6.3 loop): сканер aggro (радиус + фракция/реакция), цель угрозы (threat), генератор
  движения по `Navmesh.FindPath`, рассылка `SMSG_MONSTER_MOVE`.
- Ответный мили существа по игроку (урон игроку через систему M6.3).
- Evade/возврат на спавн + реген; «лидер» (leashing) по дистанции от спавна.

## Проверка
Живой клиент: подойти к враждебному существу → оно агрится и бежит к игроку (обходя здание),
бьёт; HP игрока падает; отбежать далеко → существо возвращается и регенит.
