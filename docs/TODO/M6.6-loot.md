# M6.6 — Лут

**Статус:** ⬜ запланировано
**Зависит от:** M6.1 (предметы/инвентарь), M6.3 (смерть существа).
**Эталон:** CMaNGOS-WotLK (`creature_loot_template`, `LootMgr`, `Loot`).

## Цель / критерий приёмки (клиент)

С убитого существа можно снять лут: труп подсвечивается (lootable), окно лута показывает
деньги/предметы, забрать предмет → в рюкзак, забрать деньги → к балансу. Пустой труп — не lootable.

## Данные
- БД `mangos`: `creature_loot_template` (entry, item, ChanceOrQuestChance, mincount/maxcount),
  деньги — `creature_template.MinLootGold/MaxLootGold` (сверить имена в схеме).

## Протокол (3.3.5a — сверить с gtker)
- Смерть → `UNIT_DYNAMIC_FLAGS |= UNIT_DYNFLAG_LOOTABLE` (values-update наблюдателям).
- `CMSG_LOOT` → `SMSG_LOOT_RESPONSE` (содержимое); `CMSG_LOOT_MONEY` → `SMSG_LOOT_CLEAR_MONEY`;
  `CMSG_AUTOSTORE_LOOT_ITEM` → выдача предмета + `SMSG_LOOT_REMOVED`; `CMSG_LOOT_RELEASE` → `SMSG_LOOT_RELEASE_RESPONSE`.

## Задачи
- Генерация лута при смерти (roll по `creature_loot_template`), хранение Loot на `WorldCreature`.
- `LootHandlers.cs`: open/money/autostore/release; начисление денег/предметов (через M6.1/M6.2 пути).

## Проверка
Живой клиент: убить существо → труп lootable → окно лута → забрать предмет (в сумке) и деньги (баланс ↑).
