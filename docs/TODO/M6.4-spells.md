# M6.4 — Спеллы (касты и эффекты)

**Статус:** ⬜ запланировано
**Зависит от:** M6.3 (бой, урон, авторитетные сущности, тик).
**Эталон:** CMaNGOS-WotLK (`Spell::cast`, `Spell::EffectSchoolDMG`, `SpellMgr`, `Spell.dbc`).

## Цель / критерий приёмки (клиент)

Персонаж кастует стартовый спелл по цели: каст-бар (если есть время каста), визуал, эффект
(урон/хил), расход маны/энергии, кулдаун. NPC получает урон от спелла / игрок лечится.

## Данные
- **`Spell.dbc`** (из клиента) — параметры спеллов (школа, эффекты, базовые значения, кост,
  каст-тайм, кулдаун, дальность). Нужен парсер DBC в `AlexWoW.DataStores` (есть пример WDBC из M5.5).
- Стартовые спеллы уже шлются (`SMSG_INITIAL_SPELLS` / `LanguageSpells`) — расширить реальными.

## Протокол (3.3.5a — сверить с gtker)
- `CMSG_CAST_SPELL` → `SMSG_SPELL_START` / `SMSG_SPELL_GO` (+ targets), `SMSG_CAST_FAILED`.
- `SMSG_SPELL_HEAL_LOG` / `SMSG_SPELLNONMELEEDAMAGELOG` (эффект урона/хила).
- `SMSG_COOLDOWN_EVENT` / `SMSG_SPELL_COOLDOWN`; power-update (`UNIT_FIELD_POWER*`).
- Авто-реген маны/энергии — в тике (M6.3 loop).

## Задачи
- Парсер `Spell.dbc` (+ кэш). Каркас `Spell`/`SpellEffectHandler` (минимум: direct damage, heal).
- `SpellHandlers.cs`: cast → валидация (дальность/ресурс/кулдаун/LoS через vmaps) → эффект → логи.
- Применение урона через систему M6.3 (`DealDamage`), хил, расход ресурса, кулдаун.

## Проверка
Живой клиент: каст спелла по NPC → урон/лог/кулдаун/мана; хил по себе → HP растёт.
