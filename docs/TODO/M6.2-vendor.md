# M6.2 — Торговля с NPC (вендор)

**Статус:** ⬜ запланировано
**Зависит от:** M6.1 (предметы, инвентарь, item-query, деньги `PLAYER_FIELD_COINAGE`).
**Эталон:** CMaNGOS-WotLK (`npc_vendor`, `Player::BuyItemFromVendor`, `Player::SellItem`).

## Цель / критерий приёмки (клиент)

У NPC-вендора открывается окно товаров; можно **купить** предмет (списываются деньги, предмет
в рюкзаке) и **продать** свой предмет (предмет уходит, деньги начисляются). Деньги персистентны.

## Данные
- БД мира `mangos`: **`npc_vendor`** (`entry, item, maxcount, incrtime, ExtendedCost`) — ассортимент
  по entry существа; цены — из `item_template.BuyPrice/SellPrice`.
- `characters`: поле денег у персонажа (`money` в таблице `characters`) + перемещение строк
  `character_items` при покупке/продаже.

## Протокол (опкоды 3.3.5a — сверить с gtker)
- `CMSG_LIST_INVENTORY` → `SMSG_LIST_INVENTORY` (список товаров вендора).
- `CMSG_BUY_ITEM` / `CMSG_BUY_ITEM_IN_SLOT` → `SMSG_BUY_ITEM` (+ `SMSG_BUY_FAILED`).
- `CMSG_SELL_ITEM` → `SMSG_SELL_ITEM` (+ failed).
- Деньги — поле `PLAYER_FIELD_COINAGE` (values-update при изменении).
- Госсип/`UNIT_NPC_FLAG_VENDOR (0x80)` уже выставляется из дампа — проверить открытие окна
  (возможно, нужен gossip-hello: `CMSG_GOSSIP_HELLO` → `SMSG_GOSSIP_MESSAGE`/прямой list).

## Задачи
- `WorldDatabase.GetVendorItemsAsync(entry)`.
- `CharactersDatabase`: деньги (чтение/запись), купля/продажа = транзакция (деньги + строки items).
- Обработчики `VendorHandlers.cs`: list/buy/sell + валидация (хватает денег/места/расстояния).
- Обновление `PLAYER_FIELD_COINAGE` и инвентаря (новые/удалённые item-объекты у клиента).

## Проверка
Живой клиент: подойти к вендору → окно товаров → купить (деньги ↓, предмет в сумке) →
продать (предмет ↓, деньги ↑) → релог: деньги/предметы сохранены.
