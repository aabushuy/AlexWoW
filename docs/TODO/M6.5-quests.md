# M6.5 — Квесты

**Статус:** ⬜ запланировано
**Зависит от:** M6.1 (предметы-награды), M6.3 (kill-цели), gossip.
**Эталон:** CMaNGOS-WotLK (`quest_template`, `creature_questrelation`, `Player::CanTakeQuest`).

## Цель / критерий приёмки (клиент)

У NPC с иконкой «!» можно взять квест, увидеть его в журнале (L), выполнить цель
(убить N существ / собрать предмет / поговорить), сдать у NPC с «?» и получить награду
(опыт/деньги/предмет). Прогресс персистентен.

## Данные
- БД `mangos`: `quest_template` (тексты, цели, награды), `creature_questrelation` /
  `creature_involvedrelation` (кто даёт/принимает).
- `characters`: таблица прогресса квестов (`character_queststatus`).

## Протокол (3.3.5a — сверить с gtker)
- `CMSG_QUESTGIVER_HELLO` / `_STATUS_QUERY` → `SMSG_QUESTGIVER_STATUS` (иконки `!`/`?`).
- `SMSG_QUESTGIVER_QUEST_LIST`, `CMSG_QUESTGIVER_QUERY_QUEST` → `SMSG_QUESTGIVER_QUEST_DETAILS`.
- `CMSG_QUESTGIVER_ACCEPT_QUEST`, `CMSG_QUESTGIVER_COMPLETE_QUEST`,
  `CMSG_QUESTGIVER_CHOOSE_REWARD` → выдача награды.
- Обновление целей: `SMSG_QUESTUPDATE_ADD_KILL` / `_ADD_ITEM`, `SMSG_QUESTUPDATE_COMPLETE`.

## Задачи
- Доступ к `quest_template`/relations; модель статуса квеста + персист.
- `QuestHandlers.cs`: hello/details/accept/complete/reward.
- Хуки прогресса: убийство существа (M6.3), получение предмета (M6.1).

## Проверка
Живой клиент: взять квест → журнал → выполнить цель → сдать → награда; релог сохраняет прогресс.
