# Дорожная карта

Проект движется **по вехам**. Каждая веха завершается *проверяемым поведением
клиента* WoW 3.3.5a — это и есть критерий приёмки.

Легенда: ✅ готово · 🟡 в работе · ⬜ запланировано

| Веха | Название | Статус |
|---|---|---|
| M0 | Каркас решения | ✅ |
| M1 | AuthServer (логин) | ✅ проверено клиентом |
| M2 | World handshake | ✅ проверено клиентом |
| M3 | Экран персонажей | ✅ проверено клиентом |
| M4 | Вход в мир | ✅ проверено клиентом (движение, чат, персистентность) |

Вехи **M0–M4** — реалистичный «играбельный минимум»: зайти в мир и побегать.
**M5–M6** — путь к полноценному играбельному серверу. **M9/M10** — глубина персонажа
(прогрессия + спелл-система). Нумерация вех — историческая (M9/M10 заведены после M6/M7/M8).

> **Текущий фокус (дорожная карта):** глубина персонажа — таланты M9.6/9.7/9.8 ✅ → **профессии
> M9.5** (следующее крупное). Веб-панель (M8) доведена до просмотра персонажей. См. раздел
> [«Дальше»](#-дальше--дорожная-карта-ближайших-срезов) внизу.

## ✅ M2. World handshake

**Критерий приёмки:** клиент проходит аутентификацию в world-сервере и
устанавливает шифрованную сессию (без вылета).

- [x] Проект `AlexWoW.WorldServer` (exe, порт 8085); сетевой код внутри проекта
- [x] Фрейминг world-пакетов (size big-endian + opcode)
- [x] **RC4 header crypt** (ключи через HMAC-SHA1 из session key) + тесты (вкл. вектор RC4)
- [x] `SMSG_AUTH_CHALLENGE`, проверено смоук-тестом по TCP
- [x] `CMSG_AUTH_SESSION` → проверка digest по session key (из auth-БД) → `SMSG_AUTH_RESPONSE`
- [x] Диспетчер опкодов (switch; handler-реестр с атрибутами — позже) + CMSG_PING/PONG
- [x] **Проверено живым клиентом** — после handshake шифрованные опкоды (0x37 CMSG_CHAR_ENUM,
      0x38C CMSG_REALM_SPLIT, 0x4FF) расшифровываются корректно → RC4 байт-в-байт
