# AlexDevCmd — клиентский пульт dev-команд (WoW 3.3.5a)

Единое окно из **3 колонок** (пропорции 1:2:4) на общей библиотеке стиля **AlexUI**:
**корни → ветви → рабочее пространство**. Навигацию и панели держит клиент; действия уходят как
dev-команды (SAY, сервер ловит `.` и не эхоит, гейт `is_admin` — на сервере).

## Колонки
- **Колонка 1 — корни:** Персонаж, Тренер, Враги, Профессия, Реагенты, Рынок, Телепорт, QA.
- **Колонка 2 — ветви** выбранного корня (или список городов для «Телепорт» / дерево категорий для «Рынок»).
- **Колонка 3 — рабочее пространство:** бесповоротная панель под выбранную ветвь (списки, поля, кнопки).

## Панели (текущий статус)
| Корень | Ветвь | Панель |
|---|---|---|
| Персонаж | Основное / Характеристики | *(Фаза 2)* — прямые сеттеры и групповой редактор статов |
| Персонаж | Бафф | наложить по id / снять по id / снять все *(список активных аур — Фаза 2)* |
| Тренер | — | заголовок «Тренер: <класс>» + «Вызвать» (`.trainer`, класс берётся клиентом) |
| Враги | Манекены | 4 манекена (`.dummy`) *(Охотник/Маг/Лечебный-70% — Фаза 2)* |
| Враги | Существа | тип + уровень + кол-во → очередь → «Запуск» (`.spawnenemy`) |

| Реагенты | — | поиск по имени → список (иконка/имя) → кол-во + «Добавить» (`.additem`) *(поиск по id — Фаза 2)* |

## Протокол
Данные приходят addon-кадрами (префикс `AlexDev`, поверх чат-опкодов 3.3.5a, в чат не печатаются):
- `devteleports` → `TBEGIN` / `T|id|faction|name` / `TEND` — список городов.
- `itemsearch|class|sub|lvlMin|lvlMax|quality|showAll|name` → `IBEGIN` / `I|id|quality|ilvl|reqlvl|name` / `IEND` — рынок/реагенты.

## Открыть/закрыть
Кнопка у миникарты или слэш **`/dev`** (`/devcmd`). Позиция окна и угол кнопки — в `AlexDevCmdDB`.

## Установка
Скопировать в `Interface\AddOns\` клиента папки **`AlexUI`** (зависимость) и **`AlexDevCmd`**, затем `/reload`.
Исходники — в репозитории (`tools/addons/`); клиентская копия отдельно. Проверки без клиента —
`tools/addons/run-checks.ps1` (см. `tools/addons/README.md`).
